# Technical Evidence (illustrative)

This folder shows how each technical finding would be **tested, evidenced, and concluded** — the auditor's procedure, the evidence requested, what "good" looks like, the sample result, and the verdict.

> **Read this first.** The underlying engagement was a **documentation-level (tabletop) assessment**, so the data below is **representative and simulated** for the fictional Northgate University environment. These are *methodology samples* — they demonstrate what evidence to request and how to evaluate it, not records of live testing against a production system. They exist to show technical fluency alongside the governance work.

Each file follows the same structure: **Objective · Evidence Requested · What "Good" Looks Like · Sample Evidence Reviewed · Finding · Audit Conclusion.**
